Delivery Lifecycle

Project delivery phases

Every project moves through one of these phases. One phase at a time. The phase tells you what kind of work is happening, not how well it's going — that's the RAG status.

1
Concept
Idea stage. Not yet assessed or committed to.
2
Not Started
Approved but no work has begun. In the queue.
3
In Planning
Scoping, scheduling, resourcing. Requirements being gathered.
4
In Execution
Work underway. Deliverables being built. Status updates happening.
5
Readiness
Testing, sign-off, go/no-go, launch prep.
6
Launch
Live. Monitoring outcomes, resolving snags, confirming success.
✓
Done
Delivered. Outcomes confirmed. Handover complete. Closed out.
Cancelled
Stopped. Will not proceed. Decision documented.
On Hold
Paused. Must have a named trigger for restart.

Status Taxonomy

RAG statuses and milestone impacts

The rule: two PMs looking at the same project should pick the same colour.

GREEN
On track. No leadership action needed.
Scope and timeline within tolerances. No open blockers or unmitigated high risks. Dependencies on track. Key decisions being made on time.
AMBER
At risk. You must state your mitigation.
Something needs attention but the project isn't off track yet. You have a plan to get it back on track and can articulate it clearly. If you can't state the mitigation, you're red.
RED
Off track. Leadership needs to know and may need to act.
A committed date will be missed without a scope or timeline change. A critical dependency has failed. Red is a signal, not a failure. The failure is staying amber when you're actually red.
COMPLETE
Delivered. Outcomes confirmed. Closed out.
Deliverables accepted by sponsor. No open actions. Handover done. Lessons learned done or scheduled.
Milestone status vs project status
Every milestone gets its own RAG. The overall project RAG is driven by the critical path. A red milestone off the critical path doesn't automatically change the project RAG — but if enough non-critical milestones slip, the float erodes. Flag when that's happening.

2.1
Meeting Notes Processing
Trigger
Paste raw notes, a transcript, or a Slack thread and ask to process meeting notes.
Output
Confluence notes page (linked), updated Action List and RAID Log, Monday updates, and a summary of every change.
What it does
Publishes a structured notes page to Confluence (summary, topic-based notes, actions table, decisions). Updates the RAID Log with new risks and dependencies. Updates the Action List, closing confirmed items and flagging overdue ones. Updates Monday.com task statuses.
2.2
Full Weekly Project Update
Trigger
"Run the weekly update" — typically at the start of the week.
Output
Updated Confluence artefacts, prioritised to-do list (must / should / flag), and a ready-to-send Slack channel post.
What it does
Scrapes configured Slack channels for the past 7 days. Updates the RAID Log, Action List, and Monday.com. Drafts this week's Confluence status page if it's Tuesday. Builds a prioritised to-do list. Drafts the Slack channel post.
2.4
Weekly Status Report (17-Section Confluence)
Trigger
"Run the weekly status", "generate this week's report", or "run the PMO skill".
Output
New versioned Confluence status page (17 sections). Slack DM to PM with the draft link and approval request.
What it does
Gathers from Slack (optional) and Confluence meeting notes. Detects RAG delta vs last week and flags any status change with a reason. Path to Green is mandatory for Amber/Red. Creates the versioned status page and DMs the PM to review and approve before marking complete.

2.3
Exec Summary / Dashboard Update
Trigger
"Give me the exec summary", "dashboard update", or "I need a brief for the sponsor".
Output
Three ready-to-paste sections: lean exec brief, dashboard portal copy, and AI insight paragraph.
What it does
Part A: Lean exec brief — TL;DR (3 bullets), decisions needed, blockers/risks, key actions. Part B: Dashboard portal copy — status, phase, sub-initiatives, risks, blockers. Part C: AI insight — core theme, 2-3 key signals, and the single most critical next step with owner.
2.10
Pre-Meeting Pack
Trigger
"Build the pre-meeting report", "build the pre-read", or "run the weekly meeting report".
Output
HTML file. Open in Chrome, click "Generate PDF", and share with the team before the sync. Added in v1.5.
What it does
Gathers from Slack, Confluence, and Monday. Builds a polished 2-page A4 landscape HTML pre-read. Page 1: objectives for today's meeting, key discussion callout, current position, timeline, and compliance status. Page 2: actions this week, top risks, sub-initiatives, workstream status, and key contacts.
2.5 + 2.6
Slack Comms
Trigger
"Give me a Slack update" (post-meeting, 2.5) or "Draft a message to [person/channel]" (ad-hoc, 2.6).
Output
Ready-to-post Slack message formatted for 30-second scanning, with named owners and clear action flags.
What it does
Post-meeting (2.5): formats meeting outcomes into a scannable Slack message with RAG status, links to status page and notes, key takeaways, and named actions. Ad-hoc (2.6): structures context, key update, risk/blocker (if any), and a clear ask — tagged as FYI, Input required, or Decision required.

2.8
RAID Extraction
Trigger
Paste any unstructured input — notes, messages, a document — and ask to extract RAIDs.
Output
Four structured tables — Blockers, Risks, Assumptions, Dependencies — with owners, impacts, and next steps.
What it does
Categorises across all four RAID dimensions. Infers owners where obvious and flags where ownership is unclear. Prioritises blockers over risks. Removes noise, consolidates duplicates, and flags items needing escalation.
2.9
Decision Extraction
Trigger
Paste any input and ask to identify or log decisions.
Output
Three decision logs: Made (owner and date), Needed (owner and required-by), Stalled (with escalation recommendation).
What it does
Extracts delivery-impacting decisions only. Distinguishes what was discussed from what was actually decided. Highlights anything blocking delivery and recommends escalation for stalled items.
2.7
Process / Governance Review
Trigger
Paste a process, ceremony, template, or governance model and ask for a review.
Output
Four-quadrant breakdown: Remove / Simplify / Automate / Keep. Async alternatives where meetings or templates are over-engineered.
What it does
Reviews against PMO principles — useful over thorough, async-first, low coordination overhead. Defaults to less. Challenges any meeting that could be a message and any template section with no clear delivery value.

Project Setup & Config

Trigger: "Populate my Confluence pages." Provide the project brief or kick-off deck and it fills the Overview, DACI, RAID Log, Actions, and Governance pages from the standard template.

Trigger: "Find my project in Confluence." Auto-discovers the full project structure — Confluence pages, Monday board, Slack channels, and key people. Runs at the start of every workflow unless a config file exists.

Trigger: "Build my project config." Produces a Project_Name_Config.md file with all page IDs, board IDs, and Slack channels. Upload to the Claude project to skip discovery on every future session.
